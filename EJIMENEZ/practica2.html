Listas y Tablas en HTML

Listas

Las listas nos sirven para ordenas y tener una mejor vista, enseguida se mostraran los tipos de listas para html

Listas no ordenadas < ul >

Las listas no ordenadas van dentro de la etiqueta < ul > y de su cierre < /ul >. Cada punto que queramos añadir a la lista, lo haremos dentro de la etiqueta < li > y su cierre. Si no le indicamos nada a la etiqueta < li >, ésta se generará de forma automática. Pero si queremos definir nosotros mismo el símbolo del punto, podemos gracias al atributo “type”.

Ejemplo

  • Hector Manzo.
  • Edsel Barbosa.
  • Eduardo Jimenez.
Para usar el "type solo lo agregamos en la parte de < li type="circle"> tambien esta square y disc que son figuras para usarlas en las listas no ordenadas.
  • Hector Manzo.
  • Edsel Barbosa.
  • Eduardo Jimenez.

Codigo Fuente

< ul >
< li type="circle">Hector Manzo.< /li >
< li type="square">Edsel Barbosa.< /li >
< li type="disc">Eduardo Jimenez.< /li >
< /ul >

Listas Ordenadas < ol >

Las listas ordenadas van enmarcadas dentro de las etiquetas < ol > < /ol >. Cada punto de la lista se escribe con la misma etiqueta que en las no numeradas: < li >. Pero al ser listas ordenadas los símbolos serán números y éstos se irán generando automáticamente por orden, conforme escribamos nuevos puntos. En las listas ordenadas podemos hacer que el primer punto comience con el número que nosotros queramos. Lo conseguiremos gracias al atributo “value”. Los siguientes puntos que escribamos se generarán automáticamente por orden, partiendo de ese número

Ejemplo

  1. Hector Manzo.
  2. Edsel Barbosa.
  3. Eduardo Jimenez.
Para usar y empezar la numeración donde usted desee debe usar el atributo "value" y el número que desee empezar, por ejemplo empezaremos desde el 5.
  1. Hector Manzo.
  2. Edsel Barbosa.
  3. Eduardo Jimenez.
Para ordenarlos con letras se pondria type="a" y para mayusculas type="A" en la parte de < ol > como lo muestra en el siguiente ejemplo:
  1. Hector Manzo.
  2. Edsel Barbosa.
  3. Eduardo Jimenez.
Para ordenarlos con numeros romanos type="i" y para mayusculas type="I" en la parte de < ol > como lo muestra en el siguiente ejemplo:
  1. Hector Manzo.
  2. Edsel Barbosa.
  3. Eduardo Jimenez.

Codigo Fuente

< ol >
< li value="20">Hector Manzo.< /li >
< li >Edsel Barbosa.< /li >
< li >Eduardo Jimenez.< /li >
< /ol >

Listas de definiciones < dl >, < dt >, < dd >.

Si lo que vamos a hacer es un listado de definiciones, podemos usar las etiquetas < dl >, < dt > y < dd >. Vamos a explicarlas por partes: La etiqueta < dl > viene de los términos ingleses “Definiton list” y nos indica que dentro de ella, entre ella y su cierre, va a ir una definición. La etiqueta < dt > viene de los términos “Definition term” y dentro de ella irá el término que vamos a definir. Para entendernos mejor, dentro de < dt > iría el título de la definición. La etiqueta < dd > viene de los términos “Definition description” y nos dice que dentro de ésta irá la definición. Si escribimos varios listados de definición, éstas se separarán automáticamente entre ellas para facilitar su diferenciación.

Ejemplo

Aquí va el término que definiremos
Y aquí dentro irá la definición propiamente dicha.
Aquí va la segunda definición
Y aquí dentro irá la segunda definición, separada automáticamente de la anterior.

Codigo Fuente

< dl >
< dt >Aquí va el término que definiremos< /dt >
< dd >Y aquí dentro irá la definición propiamente dicha.< /dd > < /dl > < dl >
< dt >Aquí va la segunda definición< /dt >
< dd >Y aquí dentro irá la segunda definición, separada automáticamente de la anterior.< /dd >
< /dl >

Tablas

Las tablas son herramientas muy útiles para presentar datos en forma de tablas y para el diseño de páginas y ubicación de textos y gráficos dentro de las mismas.

Características de las tablas

Definimos las tablas con la etiqueta < table >.
La tabla está dividida en filas definidas con la etiqueta < tr >.
Y a su vez cada fila dividida en celdas definidas con la etiqueta < td >.
Las celdas pueden contener texto, imágenes, formularios, listas, otras tablas, etc.

Sintaxis

 < table >
  < tr >
   < td >contenido celda 1< /td >
   < td >contenido celda 2< /td >
  < /tr >
 < /table >
Columna 1 Columna 2
Fila 1 Fila 2

Encabezados de las tablas

Los encabezados de una tabla se definen con la etiqueta < th > y simepre muestra los encabezados remarcados.
 < table border="1px" >
  < tr >
   < th >Nombre< /th >
   < th >Apellido< /th >
  < /tr >
  < tr >
   < td >Héctor< /td >
   < td >Manzo< /td >
  < /tr >
  < tr >
   < td >Edsel< /td >
   < td >Barbosa< /td >
  < /tr >
 < /table >

Márgenes de las celdas

Es el espacio que se encuentra entre los bordes de la celda y su contenido.Definimos los márgenes con el atributo cellpadding.
 < table border="1px" cellpadding="20px" >
  < tr >
  < th >Nombre< /th >
  < th >Apellido< /th >
  < /tr >
  < tr >
   < td >Héctor< /td >
   < td >Manzo< /td >
  < /tr >
 < /table >
Nombre Apellido
Héctor Manzo

Espaciado entre celdas

Es el espacio que se encuentra entre celda y celda.El mismo está definido con el atributo cellspacing.
 < table border="1px" cellspacing="15px" >
  < tr >
   < th >Nombre< /th >
   < th >Apellido< /th >
  < /tr >
  < tr >
   < td >Héctor< /td >
   < td >Manzo< /td >
  < /tr >
 < /table >
